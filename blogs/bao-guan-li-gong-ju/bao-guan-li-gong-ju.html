    # npm

    • NODE Package Manager，Node包管理工具
    • npm基于NODE，想要使用需要node环境
    • registry：npm包存储仓库

    # npm配置文件

    • 事实上，我们每一个项目都会有一个对应的配置文件，无论是前端项目(Vue、React )还是后端项目( Node ) ;
    • 这个配置文件会记录着你项目的名称、版本号、项目描述等;
    • 也会记录着你项目所依赖的其他库的信息和依赖库的版本号;

    生成配置文件

    npm init 
npm init -y #表示所有询问默认yes

    2

    # package.json文件

    • 实行npm init命令生成
    • 通过脚手架创建，如vue和react脚手架生成

    # 属性

    必填属性：name、version

    • name是项目的名称;
    • version是当前项目的版本号;
    • description是描述信息，很多时候是作为项目的基本描述;
    • author是作者相关信息（发布时用到);
    • license是开源协议（发布时用到) ;

    private属性

    • 记录项目是否是私有的
    • 当为true时，npm是不能发布的 禁止 npm publish

    常见属性

    main属性

    程序的入口文件

    • 这个入口和webpack打包的入口并不冲突;它是在你发布一个模块的时候会用到的;

    • 比如我们使用axios模块const axios = require('axios');

    • 实际上是找到对应的main属性查找文件的;

    • "main":"src/main.js"

    npm install

    执行npm install指令时，npm工具会从package.json的dependencies属性读取所需要下载的包

    如果我们安装一个第三方包它也依赖了其他其他包，也会写在node_modules内，且也会写入dependencies内

    • npm i axios -D：写入devDependencies
    • npm i axios -S：写入dependencies

    peerDependencies

    记录对等依赖属性

    想要使用elementplus包必须有具有vue3的包，安装时会检查peerDependencies有没有相应的包

    engin属性

    引擎配置，如指定node版本或操作系统

    browserslist属性

    浏览器配置文件，也可以创建后缀为browserslist的文件

    gitnore

    填写不git上传的文件

    # 依赖的版本管理

    npm包的版本号格式为主版本号、次版本号、修订号

    # package-lock-json

    规定死安装的版本号

    # npm安装原理

    执行安装指令时，会从npm的代码库registry下载压缩包

    实际安装时会去读取配置文件，如果发现本地缓存的包版本一致，直接从缓存拉取解压安装，如果不匹配再去服务器下载。

    如果没有lock文件

    • 会查找package.json，根据 ^ ~ 安装允许的最新的版本且更新package-lock.json

    如果已经有lock文件

    • 安装的包我们指定了版本，发现版本不一致，会下载最新的版本或指定的版本，重新构建依赖关系
    • 如果版本一致，会去查找缓存 （package-lock.json文件会记录指定安装包的索引）
    • lock文件主要目的是安装包时查找最终下载的包是否匹配本地缓存有的包，如果匹配就直接在本地缓存拿，不匹配就下载最新重构依赖关系

    3. npm uninstall package --save-dev

    # cnpm

    ​ 淘宝对registry的镜像，与国外的代码库有一定的延迟，可能没有一些较久远的版本

    ​ 一般使用cnpm工具安装来安装淘宝镜像，不直接修改npm的config
