    # pnpm创建 Vue3 + TS + Vite + Electron 项目

    1. pnpm create vite ElectronApp 创建Vite项目，选择Vue

    2. pnpm install 进入项目安装依赖

    3. pnpm install --dev electron 将electron包安装到依赖

    4.
      1
    
https://learnvue.co/2021/05/build-vue-3-desktop-apps-in-just-5-minutes-vite-electron-quick-start-guide/#building-our-vite-app

构建我们的 Vite 项目会创建一个具有以下结构的 dist 文件夹。

<img src="https://cdn.jsdelivr.net/gh/FrancisSaber/image/markdown-Image202203290101362.png" alt="img" style="zoom:50%;" />

由于我们的 Electron 代码在我们的根目录中，我们希望将整个项目的基础设置为这个 dist 文件夹。我们可以通过使用`path`库并在我们的文件中设置`base`属性来做到这一点。`vite.config.js`

    # 实现electron动态模块热重载

    思路：利用electron中的mainWindow.loadURL(<your-url>)来实现。

    对于动态模块热重载功能来说，无论是webpack还是vite，其都是将构建内容存入内存，因此我们无法使用mainWindow.loadFile('dist/index.html')这样加载文件的方式。

    但是，单纯地改变该配置也是不行的，需要使用vite将开发服务器运行起来，可以正常运行动态模块热重载，而electron直接加载其开发服务器可访问的url，即http://localhost:3000。

    1. //修改main.js内的加载方式
// mainWindow.loadFile('dist/index.html') 将该行改为下面这一行，加载url
mainWindow.loadURL("http://localhost:3000")

    3. # 同时开启vite和electron服务

      为了使vite和electron正常运行，需要先运行vite，使得其开发服务器的url可以正常访问，然后再开启electron去加载url。

      此处需要安装两个库：

      • concurrently：阻塞运行多个命令，-k参数用来清除其它已经存在或者挂掉的进程
      • wait-on：等待资源，此处用来等待url可访问
      "scripts": {
    "electron": "wait-on tcp:3000 && electron .",
    "electron:serve": "concurrently -k \"pnpm dev\" \"pnpm electron\""
},

      • yarn electron为等待tcp协议3000端口可访问，然后执行electron
      • yarn electron:serve为阻塞执行开发服务器运行和yarn electron命令

    # 打包

    想要打包应用如果main.js资源的加载方式还是URL形式是无法打包的，需要先用vite打包好，然后使用electron-builder加载vite打包后的文件进行打包。

    1. const NODE_ENV = process.env.NODE_ENV;		//获取环境变量

//根据环境渲染加载方式
win.loadURL(
    NODE_ENV === 'development'		//开发使用vite服务地址，打包选择内存地址
    ? 'http://localhost:3000'
    :`file://${path.join(__dirname, '../dist/index.html')}`
);

    3. 安装库

      • cross-env: 该库让开发者只需要注重环境变量的设置，而无需担心平台设置

      • electron-builder: electron打包库
