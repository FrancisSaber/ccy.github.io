    # CSS选择器优先级

    # 盒子模型

    盒子模型，顾名思义就是能装内容的元素。如div、li标签等。

    盒模型：分为内容（content）、填充（padding）、边框（border）、边界（margin）四个部分，内容又分为高(height)、宽(width)。

    div 高(height)默认为auto，会由子元素的改好、宽(width)。

    # 盒子模型的类型

    • W3C标准盒模型：W3C 标准盒模型：属性 width ，height 只包含内容 content，不包含 border 和 padding 。
    • IE盒模型：属性width，height包含content、border和padding，指的是content + padding + border

    # 如何切换

    • div{
    box-sizing:content-box;		/*默认*/
    box-sizing:border-box;
}

    # margin值为负数

    • 对于margin-top和margin-left，这两个属性会让盒子分别上移和左移，且会影响其他元素。
    • 对于margin-right和marign-bottom，当为负值时自身不会移动，但会让盒子右方和下方的元素向盒子移动。相当于对右方和下方元素来说，自身高度消失，因此它们可以移动覆盖。(可以利用此特性实现圣杯布局)

    # BFC

    # 什么是BFC

    ​ 块级格式化上下文。BFC它决定了元素如何对其内容进行定位，以及与其它元素的关系和相互作用，当涉及到可视化布局时，Block Formatting Context提供了一个环境，HTML在这个环境中按照一定的规则进行布局。简单来说块级BFC实现了独立了布局空间，让BFC的内容不会影响到外部布局。

    # 如何触发

    • overflow: hidden
    • display: inline-block
    • position: absolute
    • position: fixed
    • display: table-cell
    • display: flex

    # BFC的规则

    • BFC就是一个块级元素，块级元素会在垂直方向一个接一个的排列
    • BFC就是页面中的一个隔离的独立容器，容器里的标签不会影响到外部标签
    • 垂直方向的距离由margin决定， 属于同一个BFC的两个相邻的标签外边距会发生重叠
    • 计算BFC的高度时，浮动元素也参与计算

    # 网格布局-gird

    Grid 布局即网格布局，是一种新的 CSS 布局模型，比较擅长将一个页面划分为几个主要区域，以及定义这些区域的大小、位置、层次等关系。实现了CSS 二维布局。

    # 网格布局和弹性布局的区别

    弹性布局只能处理一个维度的布局（即一次只能处理一行或一列的布局）。网格布局将容器分成了行和列。产生了一个个网格，通过将内容放在网格内实现更加强大的布局。

    # minmax(函数)

    我们有时候想给网格元素一个最小和最大的尺寸，minmax() 函数产生一个长度范围，表示长度就在这个范围之中都可以应用到网格项目中。它接受两个参数，分别为最小值和最大值。grid-template-columns: 1fr 1fr minmax(300px, 2fr) 的意思是，第三个列宽最少也是要 300px，但是最大不能大于第一第二列宽的两倍。代码以及效果如下：

  grid-template-columns: 1fr 1fr minmax(300px, 2fr);
  grid-gap: 5px;
  grid-auto-rows: 50px;

    # flex

    flex为三个属性的统一，分别是flex-grow、flex-shrink、flex-basis。flex默认值值是：0 1 auto，即不放大、自动按比例缩小、；常见的flex：1是 1 1 0%，即不放大、不缩小、根据内容展开；

    flex:n n单独设置flex-grow属性

    flex-grow

    flex-grow：规定项目的放大比例，即flex容器不换行的情况下且行内还要剩余空间，如果该属性为0，不放大。如果所有项目的flex-grow都为1，等比例放大。

    如果一个项目的flex-grow属性为2，其他项目都为1，则前者占据的剩余空间将比其他项多一倍。

    • 弹性容器的宽度正好等于元素宽度总和，无多余宽度，此时无论flex-grow是什么值都不会生效

    flex-shrink

    定义了项目的缩小比例。如果该属性为0，则不会缩小。如果都为1，按比例缩小。

    • 该属性生效的前提是容器宽度不够分

    flex-basis

    flex-basis设置的是元素在主轴上的初始尺寸，所谓的初始尺寸就是元素在flex-grow和flex-shrink生效前的尺寸。浏览器根据这个属性，计算主轴是否有多余空间，默认值为auto，即项目的本来大小，如设置了width则元素尺寸由width/height决定（主轴方向），没有设置则由内容决定。当设置为0的是，会根据内容撑开。

    # order

    当个项目的排列序号，数值越小越前。

    # align-self

    对flex容器内的单个项目在交叉轴方向对齐方式进行设定。

    # css实现三角形

    方法：设置元素宽度高度为0，设置border大小，将三个border的颜色设为透明，留下一个border有颜色。实际效果是一个由四个等边三角形组成的正方形，只不过其他3个三角形背景透明因此看上去只有一个三角型。

    border-bottom:20px solid red;		/*边框颜色透明 solid transparent*/

    # px、em和rem

    # 区别

    px是固定的像素，是相对于显示器屏幕分辨率而言的，一旦设置了就无法因为适应页面大小而改变。

    em是相对于当前对象内文本的字体尺寸，若没有设置行内文本的字体尺寸，则使用浏览器默认字体尺寸，一般为16px。

    rem是CSS3新增加的相对单位，即root em，是相较于根元素（html)。rem的根据根节点使得计算更清晰。

    em 和 rem 单位之间的区别是浏览器根据谁来转化成px值，理解这种差异是决定何时使用哪个单元的关键。

    em像素值等于em值乘以使用em单位的元素的字体大小。

    W3C标志规定em是相对于使用em单元的元素的字体大小

    rem实现只修改根元素就能成比例地调整所有字体的大小。

    # 伪类和伪元素

    伪类和伪元素都是用来表示文档树之外的元素，伪类用单冒号、伪元素用双冒号。核心区别在于是否创造了新的元素。

    伪类其核心就是用来选择那些不能够被普通选择器选择的文档之外的元素，通俗来说就是表示元素的某种状态，比如hover元素被覆盖的这个状态，我们是无法通过元素的ID、或类来获取元素的特殊状态。

    伪元素表示被选择元素的某个部分，这个部分的效果像一个独立的函数，但其实际上不存在与源文档中，只存在于css，如::before

    # 常见伪类

    • hover、focus、first-child、last-child

    # 常见伪元素

    • ::before、after、first-line（第一行）、first-letter（第一个字）
    • ::section、placeholder css3新加

    可以利用伪元素进行清除浮动

    # display: inline-block的问题

    display可以设置BFC

    当在一行中需要展示多个拥有块级属性的标签元素时，通常选择display:inline-block;

    优点：不用设置浮动或定位，浮动脱离文档流还需要清除浮动，定位降低扩展性。

    问题：

    1、标签元素之间会出现间隙（原因：inline-block拥有默认字体大小属性，包括标签的换行）；

    2、当标签内容为空时，会造成布局错乱（原因：inline-block默认字体对齐方式是vertical-align:baseline;即底部对齐）；

    解决：

    间隙处理：

    1、给父级元素设置定位；

    2、元素标签不换行；

    3、父级元素的font-size:0;清除字体影响；

    4、浮动；-

    布局错乱：

    1、浮动；

    2、定位；

    3、设置verticle-align:top;字体上对齐；

    # Span标签

    # span标签有这么几个特点:

    1、span标签是一个内联元素,只能容纳文本或者其他内联元素；

    2、和其他内联元素一样,内联元素的宽度和高度就是包含的文字或图片的宽度和高度

    3、给span标签设置高height、宽width、上下内边距(padding-top/bottom)、上下外边距(margin-top/bottom)都是不起作用的。

    给span标签设置左右内边距(padding-left/right)、左右外边距(margin-left/right)、行高(line-height)是有效的。
