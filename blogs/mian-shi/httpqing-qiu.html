    # Ajax

    ​ ajax是一种异步通信方式，通过直接由JavaScript脚本向服务器发起http请求。根据服务器返回的内容，网页能够快速地增量呈现更新的响应部分，不用重载网页。

    # 优点

    • 只需对网页进行局部更新，无需刷新整个网页，节省带宽，增强用户体验
    • 支持jsonp请求

    # 缺点

    • 属于原生js，基于XHR开发

    # 使用

    var xhr = new XMLHttpRequest();
xhr.setRequestHeader();			//设置请求头信息
xhr.open('GET', url);			//有第三个参数，默认为false表示是否同步方式执行，部分浏览器版本不支持
xhr.onreadystatechange = function() {	//监听函数
    if (xhr.readyState === 4 && xhr.status === 200) {
        console.log(xhr.responseText)   // 从服务器获取数据
    }
}
xhr.send();		//发起请求

    # fetch

    ​ fetch号称是ajax的替代品，它的API是基于Promise设计的，旧版本的浏览器不支持Promise，需要使用polyfill es6-promise

    不支持超时控制

    ​ fetch() 必须接受一个参数——资源的路径。无论请求成功与否，它都返回一个 Promise 对象，resolve 对应请求的 Response (opens new window)。

    # 优点

    • fetch是原生js，脱离了XHR
    • 提供丰富的API，调用简单
    • 解决回调地狱的问题，返回的是一个promise对象，可以链式调用

    # 缺点

    • 部分浏览器不支持l
    • 终止取消不方便
    • 默认不带cookie，需要设置credentials
    • 不支持进度控制、无法取消或超时处理

    # 使用

    fetch(url, {
    methods:'POST',
    header:{},
    body: ...
})
    .then(response => {
        if (response.ok) {
            return response.json();
        }
    })
    .then(data => console.log(data))
    .catch(err => console.log(err))

//配合 async和await使用
async function fetchData() {
    let respones = await fetch(url);
    let data = await respones.json();
}

    # 区别

    ​ 当接收到一个代表错误的 HTTP 状态码时，从 fetch()返回的 Promise 不会被标记为 reject， 即使该 HTTP 响应的状态码是 404 或 500。相反，它会将 Promise 状态标记为 resolve （但是会将 resolve 的返回值的 ok 属性设置为 false ）， 仅当网络故障时或请求被阻止时，才会标记为 reject。 默认情况下, fetch 不会从服务端发送或接收任何 cookies, 如果站点依赖于用户 session，则会导致未经认证的请求（要发送 cookies，必须设置 credentials 选项）表示用户是否在跨域请求的情况下从其他域发送cookies

    # axios

    axios也是一种异步通信方式，是对原生XHR请求的封装， 可实现取消请求，超时处理，进度处理等等

    • 可以在node.js中使用
    • 提供了并发请求的接口
    • 支持Promise API
    • 浏览器兼容性好
    • 有拦截器可以进行统一处理
    • 可自动将数据转为json格式

    /*
	取消方法1 使用 CancelToken.source 工厂方法创建 cancel token：
	axios取消请求与Fetch相似，获取axios对象的CancelToken
	在发起axios请求时将source的token作为第二个参数对象的cancelToken值传入
	想要取消执行source.cancel()方法
*/

const CancelToken = axios.CancelToken;		
const source = CancelToken.source();

axios.get('/user/12345', {
  cancelToken: source.token
}).catch(function(thrown) {
  if (axios.isCancel(thrown)) {
    console.log('Request canceled', thrown.message);
  } else {
     // 处理错误
  }
});

axios.post('/user/12345', {
  name: 'new name'
}, {
  cancelToken: source.token
})

// 取消请求（message 参数是可选的）
source.cancel('Operation canceled by the user.');

    # axios取消重复请求

    /*
	思路 一下遇到重复的请求时删除上一个请求，执行最新的
	1.可以先设置个map存储请求的标识和取消方法
	2.请求标识key可以根据请求的方法、url和参数来唯一标识
	3.对请求进行拦截，请求前判断先获取请求的标识
	4.判断map是否已经存在相同的请求，如果存在相同的请求进行取消，从map获取相应的取消方法并执行并删除其在map的记录
	5.如果不存在则存储其key和取消方法
	6.在响应拦截器删除对应的请求在map的记录
*/
/*
	如果只想执行最开始的请求可以在请求拦截器直接执行对应的cancel方法
*/

    import axios from 'axios'
import baseURL from './config'
import qs from 'qs'

const pendingRequest = new Map(); // 请求对象
const CancelToken = axios.CancelToken;

axios.defaults.timeout = 30000
axios.defaults.baseURL = baseURL.target

// 添加请求拦截器
axios.interceptors.request.use(function(config) {
  // 在发送请求之前做些什么
  config.headers = {
      'content-type': 'application/json',
      'token': getToken()
  }
  // 获取请求key
  let requestKey = getReqKey(config);

  // 判断是否是重复请求，对于最新的请求不做处理，将会在response里拦截处理
  if (pendingRequest.has(requestKey)) { // 是重复请求
    removeReqKey(requestKey); 		   // 同时会取消并删去map的记录
  } else{
    // 设置cancelToken
    config.cancelToken = new CancelToken(function executor(cancel) {
      pendingRequest.set(requestKey, cancel); // 设置
    })
  }

  return config;
}, function (error) {
  // 请求错误
  return Promise.reject(error);
});

// 添加响应拦截器
axios.interceptors.response.use(function (response) {
  // 请求对象中删除requestKey
  let requestKey = getReqKey(response.config);
  removeReqKey(requestKey);

  // 对返回数据做点啥 比如状态进行拦截
  if (response.data.status !== 200) {
      Toast({
        message: response.data.message,
        type: 'warning',
        duration: 1000
      })
      return
    }
    
  // 没问题 返回服务器数据
  return response.data;
}, function (error) {
  let requestKey = getReqKey(error.config);
  removeReqKey(requestKey);
  
  // 响应错误
  return Promise.reject(error);
});

// 获取请求key
function getReqKey(config) {
  // 请求方式、请求地址、请求参数生成的字符串来作为是否重复请求的依据
  const { method, url, params, data } = config; // 解构出来这些参数
  // GET ---> params  POST ---> data
  const requestKey =  [ method, url, qs.stringify(params), qs.stringify(data)].join('&');
  return requestKey;
}

// 取消重复请求
function removeReqKey(key) {
    if(pendingRequest.has(key)) {
        const cancelToken = pendingRequest.get(key);
        cancelToken(key); // 取消之前发送的请求
        pendingRequest.delete(key); // 请求对象中删除requestKey
    }
}
