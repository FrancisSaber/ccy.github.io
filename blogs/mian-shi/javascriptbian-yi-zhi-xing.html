    JavaScript代码的执行

    月夕花辰 2022/3/17 JavaScript2

    # V8处理js代码流程

    • 浏览器将JavaScript代码交给V8引擎，先进行的是编码格式转换，转成UTF-16.

    • 接着对代码进行词法分析，分别成有意义的tokens

    • 对tokens进行语法分析转成AST树，大部分JavaScript引擎为了减少内存的使用，不会都整个JavaScript代码都解析成功抽象语法树，而是有个lazy-parsing的过程，对于不是立即执行的函数，会进行预解析。

    • 将tokens解析成AST树后，引擎的ignition会负责将AST解释成字节码，最后再转成机器码，同时ignition会标记热点函数，到函数的执行次数达到一定阀值后，会交由turbofan的编译器直接解析成优化后的机器码（这个叫jit（即时编译）），如果热点函数执行的内容发生变化，会交由deoptimition反编译成字节码

    ​ 早期谷歌的JavaScript引擎是直接将AST解释成机器码的，但由于需要占用过多的内存因此后来改为字节码（字节码的好处之一是可以跨平台，不同CPU的机器码可能不同）

    # 概念

    # 静态作用域

    JavaScript采用的是词法作用域（静态作用域），其作用域范围在其定义时就已经确定了。

    # 执行上下文栈

    JavaScript代码想要执行，必须进入执行栈内。栈内存储的是执行上下文。栈有一个扩展栈指针寄存器来指向栈顶。通过上下文的弹入弹出工作。

    # 执行上下文

    对于JavaScript代码，分为全局代码、函数代码、eval代码；当执行JavaScript代码前，需要进行准备工作即创建执行上下文。

    每个执行上下文包含三个重要属性

    • VO（变量对象），对于函数而言是活动对象AO
    • 作用域链：
    • this

    # 执行上下文生命周期

    1. 创建执行上下文
    2. 代码执行
    3. 回收阶段（GC）

    # 执行过程

    # 上下文初始化

    当创建好执行上下文初始化后，对于全局上下文，变量对象初始化是全局对象；对于函数上下文，变量对象初始化只包括argument类数组对象。

    # 进入执行上下文

    在进入执行上下文后会给变量对象添加属性对属性初始化，此时变量对象包括变量声名、函数声名，对于函数执行上下文，AO对象还包括形式参数值被赋为undefined和一个可迭代的类数组对象argument。这个过程也就是常说的函数预编译、变量提升的过程。

    在函数调用，进入执行上下文，创建好VO、AO后，会将活动对象添加到作用域链前

    function foo() {
	function bar() {}

    globalContext.VO
]
bar.[[scope]] = [
    fooContext.VO

    # 最新ECMA标准

    在ES5+之后，执行上下文的内容变成This Binding、词法环境、变量环境。

  LexicalEnvironment = { ... },
  VariableEnvironment = { ... },

    为了继续去适配早期的JS的var等，新的规范增加了变量环境（VariableEnvironment）。变量环境也是⼀个词法环境，其环境记录器包含了由变量声明语句

    在ES6 中，词法环境组件和变量环境组件的区别在与前者⽤于存储函数声明和变量（ let 和 const ）绑定，⽽后者仅⽤与存储变量（ var ）绑定。

    具体例子

    let a = 20; 
const b = 30; 
var c; 
function multiply(e, f) {
    var g = 20; 
    return e * f * g; 
} 
c = multiply(20, 30);

    # 分析程序执行全过程

    # 全局代码

    • 程序启动，全局执行上下文被创建，压入调用栈
      1. 创建全局上下文的 词法环境
        1. 创建 对象环境记录器 ，它用来定义出现在 全局上下文 中的变量和函数的关系（负责处理 let 和 const 定义的变量）
        2. 创建 外部环境引用，值为 null
      2. 创建全局上下文的 变量环境
        1. 创建 对象环境记录器，它持有 变量声明语句 在执行上下文中创建的绑定关系（负责处理 var 定义的变量，初始值为 undefined 造成声明提升）
        2. 创建 外部环境引用，值为 null
      3. 确定 this 值为全局对象（以浏览器为例，就是 window ）

    # 函数被调用

    • 函数被调用，函数执行上下文被创建，压入调用栈
      1. 创建函数上下文的 词法环境
        1. 创建 声明式环境记录器 ，存储变量、函数和参数，它包含了一个传递给函数的 arguments 对象（此对象存储索引和参数的映射）和传递给函数的参数的 length。（负责处理 let 和 const 定义的变量）
        2. 创建 外部环境引用，值为全局对象，或者为父级词法环境（作用域）
      2. 创建函数上下文的 变量环境
        1. 创建 声明式环境记录器 ，存储变量、函数和参数，它包含了一个传递给函数的 arguments 对象（此对象存储索引和参数的映射）和传递给函数的参数的 length。（负责处理 var 定义的变量，初始值为 undefined 造成声明提升）
        2. 创建 外部环境引用，值为全局对象，或者为父级词法环境（作用域）
      3. 确定 this 值
    • 进入函数执行上下文的执行阶段：
      1. 在上下文中运行/解释函数代码，并在代码逐行执行时分配变量值。
