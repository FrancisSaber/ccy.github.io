    过程

    月夕花辰

    # 过程

    1. Vue实例化过程会执行_init()进行初始化，会先执行initState将data数据进行拦截
    2. 在进行一系列初始化后，Vue会通过$mount会定义一个updateComponet方法执行render渲染函数生成虚拟节点，通过update方法生成真实的DOM节点
    3. 通过new Watcher执行这个updateCompoent方法，这个过程会返回到响应式数据从而进行依赖收集。

    # 依赖收集

    在进行数据拦截的过程中，Vue会为每个响应式数据实例化一个dep对象，用于依赖收集。

    当new watcher，回去获取想要侦测的数据，对于Vue实例是updateComponent方法，内部是先通过get方法去获取返回对象，同时会将Watcher实例通过pushTarget赋给Dep.target，在响应式数据的get方法触发时执行dep.depend将dep实例加到watcher的dep数组中，同时dep的也会收集watcher实例。这是个双向保存的过程。

    1. 先watcher收集dep实例
    2. 再dep收集watcher实例

    Dep类维护了一个栈在一个新watcher创建时入栈，一个watcher收集完成出栈，实时更新Dep.target指向的watcher

    class Dep{
    constructor() {
        this.subs = [];
    }
    depend() {
        if(Dep.targer) {
            Dep.target.addDep(this);
        }
    }
    let stack = [];
    addSub(watcher) {
		this.subs.push(watcher)
    }
    function pushTarget(watcher) {
        Dep.target = watcher
        stack.push(watcher)
    }
    // 运行完之后清空当前的watcher
    function popTarget() {
        stack.pop()
        Dep.target = stack[stack.length - 1]
    }
}

    class Watcher{
    constructor(vm, exprOrFn, cb, options, isRenderWatcher) {
        this.deps = [];
        this.getter = exprOrFn;
        this.value = this.get()
    }
    get() {
        pushTarget(this);
        let value = this.getter.call(vm, vm);
        popTarget()
        return value;
    }
    //dep不会重复收集watcher，
    addDep(dep) {
        this.deps.push(dep);//将dep实例加入到watcherdeps内
        dep.addSub(this);	//将wathcer加入到dep内
    }
    update() {				//dep通知watcher执行update方法就是重新获取数据
		this.get();
    }
}

    # computer

    1. 从options获取computed对象
    2. 创建一个watcher空对象
    3. 遍历computed对象，获取每个computed属性的值（需要判断是函数还是对象）
    4. 为每个属性new一个watcher挂载到watcher对象上，同时会传入lazy配置表示缓存信息同时也可以区分computed和普通watcher
    function initComputed(vm) {
    const computed = vm.$options.computed;
    const watchers = vm._computedWatcher = Object.create(null);
    for(const key in computed) {
        const userDef = computed[key];
        const getter = typeof userDef === 'function' ? userDef : userDef.get
        watchers[key] = new Watcher(vm, getter, noop, {lazy: true})
        if (!(key in vm)) {
            defineComputed(vm, key, userDef)
        }
    }
}

    function defineComputed(vm, key, useDef) {
    let getter = null;
    if(typeof useDef === 'function') {
        getter = createComputedGetter(key);
    } else {
        getter = useDef.get
    }
    //通过Object.defineProperty挂载computed
    Object.defineProperty(vm, key, {
        get: getter,
        set: function() {}
    })
    function createComputedGetter() {
        return computedGetter() {
            const watcher = this._computedWatcher[key];
            if(watcher) {
                if(watcher.dirty) watcher.evluate()
            }
            if(Dep.target) {
                watcher.depend();
            }
            return watcher.value;
        }
    }
}

    当dirty为true才会重新计算，重新计算会将dirty再次置为false表示缓存。当计算属性所用到的值修改时，会执行计算属性wathcer的update方法，将重新计算并将dirty置为true，由于渲染watcher也会收到通知，因此会重新渲染。

    当渲染函数重新获取计算属性时会如果dirty为false就不会执行watcher.evluate而是直接返回value

    class Watcher{
	evluate() {
        this.value = this.get();
        this.dirty = false;
    }
    get() {
        const vm = this.vm
        //修改Dep.target指向
        pushTarget(this)
        // 执行函数
        let value = this.getter.call(vm, vm)
        //computed watcher收集完毕后出栈
        popTarget()
        return value
    }
    //让计算属性用到的状态的dep实例收集渲染函数
    depend() {
        let i = this.deps.length;
        while(i--) this.deps.depend();	
    }
    update(){
        if (this.lazy) {
            this.dirty = true
        } else {
            this.get()
        }
    }
}

    computed流程

    ​ 首先render渲染过程会读取计算属性值，这个值是直接在vm实例上获取，因为在computed初始化时就已经通过createComputedGetter方法在vm上挂载，首先是判断watcer的dirty是否为true是的话通过watcher.evluate()计算值赋给watcher的value，再判断是否存在Dep.target此时指向计算属性的watcher，将计算属性的watcher加入到每个dep实例的subs内。

    1、首先在render函数里面会读取this.info，这个会触发createComputedGetter(key)中的computedGetter(key)；

    2、然后会判断watcher.dirty，执行watcher.evaluate()；

    3、进到watcher.evaluate()，才真想执行this.get方法，这时候会执行pushTarget(this)把当前的computed watcher push到stack里面去，并且把Dep.target 设置成当前的computed watcher`；

    4、然后运行this.getter.call(vm, vm) 相当于运行computed的info: function() { return this.name + this.age }，这个方法；

    5、info函数里面会读取到this.name，这时候就会触发数据响应式Object.defineProperty.get的方法，这里name会进行依赖收集，把watcer收集到对应的dep上面；并且返回name = '张三'的值，age收集同理；

    6、依赖收集完毕之后执行popTarget()，把当前的computed watcher从栈清除，返回计算后的值('张三+10')，并且this.dirty = false；

    7、watcher.evaluate()执行完毕之后，就会判断Dep.target 是不是true，如果有就代表还有渲染watcher，就执行watcher.depend()，然后让watcher里面的deps都收集渲染watcher，这就是双向保存的优势。

    8、此时name都收集了computed watcher 和 渲染watcher。那么设置name的时候都会去更新执行watcher.update()

    9、如果是computed watcher的话不会重新执行一遍只会把this.dirty 设置成 true，如果数据变化的时候再执行watcher.evaluate()进行info更新，没有变化的的话this.dirty 就是false，不会执行info方法。这就是computed缓存机制。
