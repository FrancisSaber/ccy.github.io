    # clientRootMixin （根组件混入）

    指向 mixin 文件的路径，它让你可以控制根组件的生命周期：

    // 插件的入口
const path = require('path')

module.exports = {
  clientRootMixin: path.resolve(__dirname, 'mixin.js')
}
// mixin.js
export default {
  created () {
      console.log('插件在根组件创建时进行操作')
  },
  mounted () {
      console.log('插件可以在根组件挂载进行操作')
  }
}

    # enhanceAppFiles

    此选项接受指向增强文件的绝对文件路径或返回该路径的函数，你可以通过此选项做一些应用级别的配置 (opens new window):

    //	./lib/enhanceAppFile.js'
import Live2D from "./live2D.vue"
// import * as config from "../package.json"
const config = require('../package.json');

export default ({
  Vue
}) => {
  // 自定义扩展参数
  Vue.prototype.$kanbanLive2dAppName = config.name;
  Vue.prototype.$kanbanLive2dAppUrl = config.homepage;
  Vue.prototype.$kanbanLive2dAppDescription = config.description;

  // 注册Vue全局组件
  Vue.component('Live2D', Live2D);
}

// index.js
module.exports = (options, context) => ({
    enhanceAppFiles: [
        path.resolve(__dirname, './lib/enhanceAppFile.js'),
        path.resolve(__dirname, './....')		//在组件初始化时进行操作
    ],
	//enhanceAppFiles 此选项接受指向增强文件的绝对文件路径或返回该路径的函数
    //enhanceAppFiles 通过此选项创建注册看板娘组件返回路径函数，并通过globalUIComponents选项往全局注册
    globalUIComponents: ['Live2D'],			
})
