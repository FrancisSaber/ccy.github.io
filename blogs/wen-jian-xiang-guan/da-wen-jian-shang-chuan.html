    # JavaScript文件操作相关概念

    # Blob

    Blob 对象表示一个不可变、原始数据的类文件对象。它的数据可以按文本或二进制的格式进行读取，也可以转换成 ReadableStream (opens new window) 来用于数据操作。

    Blob 表示的不一定是JavaScript原生格式的数据。File (opens new window) 接口基于Blob，继承了 blob 的功能并将其扩展使其支持用户系统上的文件。

    要从其他非blob对象和数据构造一个 Blob，请使用 Blob() (opens new window) 构造函数。要创建一个 blob 数据的子集 blob，请使用 slice() (opens new window) 方法。要获取用户文件系统上的文件对应的 Blob 对象，可以通过File接口。

    方法

    **Blob.slice()**方法用于创建一个包含源 Blob (opens new window)的指定字节范围内的数据的新 Blob (opens new window) 对象。

    var blob = instanceOfBlob.slice([start [, end [, contentType]]]};
    1

    start：表示第一个会被会被拷贝进新的 Blob (opens new window) 的字节的起始位置。

    end：这个参数代表的是 Blob (opens new window) 的一个下标，这个下标-1的对应的字节将会是被拷贝进新的Blob (opens new window) 的最后一个字节。如果你传入了一个负数，那么这个偏移量将会从数据的末尾从后到前开始计算。举例来说， -10 将会是 Blob (opens new window) 的倒数第十个字节。它的默认值就是它的原始长度。

    contentType： 可选给新的 Blob (opens new window) 赋予一个新的文档类型。这将会把它的 type 属性设为被传入的值。它的默认值是一个空的字符串。

    示例

    使用Blob创建一个指向类型化数组的URL

    var typedArray = GetTheTypedArraySomehow();
var blob = new Blob([typedArray.buffer], {
    type: 'application/octet-stream'
}); 
// 传入一个合适的 MIME 类型
var url = URL.createObjectURL(blob);
// 会产生一个类似 blob:d3958f5c-0777-0845-9dcf-2cb28783acaf 这样的URL字符串
// 你可以像使用普通 URL 那样使用它，比如用在 img.src 上。

    # File

    文件（File）接口提供有关文件的信息，并允许网页中的 JavaScript 访问其内容。

    通常情况下， File 对象是来自用户在一个<input>元素上选择文件后返回的 FileList (opens new window) 对象,也可以是来自由拖放操作生成的 DataTransfer (opens new window) 对象，或者来自 HTMLCanvasElement (opens new window) 上的 mozGetAsFile() API。

    # File 构造函数

    //返回一个新构建的文件对象（File）。
var file = new File(["foo"], "foo.txt", {
  type: "text/plain",
});

    参数

    bits：一个包含ArrayBuffer (opens new window)，ArrayBufferView (opens new window)，Blob (opens new window)，或者 DOMString (opens new window) 对象的 Array (opens new window) — 或者任何这些对象的组合。这是 UTF-8 编码的文件内容。

    name：表示文件名称，或者文件路径。

    options ：可选参数，选项对象，包含文件的可选属性。可用的选项如下：

    • type: DOMString (opens new window)，表示将要放到文件中的内容的 MIME 类型。默认值为 "" 。
    • lastModified: 数值，表示文件最后修改时间的 Unix 时间戳（毫秒）。默认值为 Date.now() (opens new window)。

    # 大文件上传思路

    问题 1：谁负责资源分块？谁负责资源整合？

    当然这个问题也很简单，肯定是前端负责分块，服务端负责整合.

    问题 2：前端怎么对资源进行分块？

    首先是选择上传的文件资源，接着就可以得到对应的文件对象 File，而 File.prototype.slice 方法可以实现资源的分块，当然也有人说是 Blob.prototype.slice 方法，因为 Blob.prototype.slice === File.prototype.slice.

    问题 3：服务端怎么知道什么时候要整合资源？如何保证资源整合的有序性？

    由于前端会将资源分块，然后单独发送请求，也就是说，原来 1 个文件对应 1 个上传请求，现在可能会变成 1 个文件对应 n 个上传请求，所以前端可以基于 Promise.all 将这多个接口整合，上传完成在发送一个合并的请求，通知服务端进行合并。

    合并时可通过 nodejs 中的读写流（readStream/writeStream），将所有切片的流通过管道（pipe）输入最终文件的流中。

    在发送请求资源时，前端会定好每个文件对应的序号，并将当前分块、序号以及文件 hash 等信息一起发送给服务端，服务端在进行合并时，通过序号进行依次合并即可。

    问题 4：如果某个分块的上传请求失败了，怎么办？

    一旦服务端某个上传请求失败，会返回当前分块失败的信息，其中会包含文件名称、文件 hash、分块大小以及分块序号等，前端拿到这些信息后可以进行重传，同时考虑此时是否需要将 Promise.all 替换为 Promise.allSettled 更方便.

    # 上传接口

    import axios from "axios";

const baseURL = 'http://localhost:3001';
//上传单个分块接口
export const uploadFile = (url, formData, onUploadProgress) => {
    return axios({
        methods:'post',
        url,
        baseUrl,
        headers: {
            'Conten-type': 'multipart/form-data'
        },
        data: formData,
        onUploadProgress,         //回调设置每个分块进度
    })
}

//通知服务端合并分块接口
export const mergeChunk = (url, data) => {
    return axios({
        methods:'post',
        url,
        baseURL,
        headers: {
          'Content-Type': 'application/json'
        },
        data
    })
}

	通过type为file的input标签的点击事件拿到源文件对象
	<input type="file" @change="fileChange" />
*/
let currFile;		//源文件对象
let fileChunkFiles = [];		//存储分块的数组
const fileChange = async (event) {
    const [file] = event.target.files;
    if(!file) return;
    currFile = file;
    fileChunkFiles = await getFileChunk(file);    //根据设定值切块，返回一个hash值文件
    uploadChunks(fileHash);     
}

const uploadChunks = (fileHash) => {
    // requests 将拿到每个上传分块的promise对象
    const requests = fileChunkList.map((item, index) => {
        const formData = new FormData();
        //设置分块文件信息
        formData.append(`${currFile.value.name}-${fileHash}-${index}`, item.chunk);
        formData.append("filename", currFile.value.name);
        formData.append("hash", `${fileHash}-${index}`);        //设置每个分块的hash加上序号
        formData.append("fileHash", fileHash);                  //每个分块设置同一个文件hash，表示是同一个文件
        return uploadFile('/upload', formData, onUploadProgress(item));   //每个分块上传情况
    }); 
    Promise.all(requests).then(() => {
        mergeChunks('/mergeChunks', {
            size: DefualtChunkSize, 			//默认分块大小 5 * 1024 * 1024  5M
            filename: currFile.name        //文件名
        })
    })
}

const uploadFile = (file, chunkSize = DefualtChunkSize) => {	// 5 * 1024 * 1024
    return new Promise((resolve) => {
        let blobSlice = File.prototype.slice || File.prototype.mozSlice || File.prototype.webkitSlice;
        let chunks = Math.ceil(file.size / chunkSize); //Math.ceil() 函数返回大于或等于一个给定数字的最小整数。
        let currentChunk = 0;
        function loadNext() {
            if (chunks > urrentChunk) return;		//当分块上传完成结束
            let start = currentChunks * chunkSize;
            let end = ((start + chunkSize) >= file.size) ? file.size : start + chunkSize;
            let chunk = blobSlice.call(file, start, end);
            fileChunkList.push({
                chunk,
                size: chunk.size,
                name: currFile.name
            })
            currentChunk++;
            loadNext();
        }
        loadNext();
    })
}

    # 计算总进度条

    const totalPercentage = computed(() => {            //根据上传进度不断计算更新
    if (!fileChunkList.value.length) return 0;
    const loaded = fileChunkList.value
        .map(item => item.size * item.percentage)
        .reduce((curr, next) => curr + next);
    return parseInt((loaded / currFile.value.size).toFixed(2));   //保留两位小数
})

const onUploadProgress = (item) => (e) => {
