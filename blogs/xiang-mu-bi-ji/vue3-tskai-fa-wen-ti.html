    # ts声名数组

    ​ TypeScript具有ReadonlyArray<T>类型，它与Array<T>相似，只是把所有可变方法去掉了，因此可以确保数组创建后再也不能被修改：

    let a: number[] = [1, 2, 3, 4];
let ro: ReadonlyArray<number> = a;
ro[0] = 12; // error!
ro.push(5); // error!
ro.length = 100; // error!
a = ro; // error!

    # 对话框内的组件通过props传值，打开后对话框的表单数据没有成功渲染

    //先前写法
//将要传值的props先传了过去。再打开对话框，这导致了组件此时还未加载，就开始传值因此导致数据渲染失败
propsValue = value
dialogVisible = true
//应该换成如下写法
dialogVisible = true
nextTick(() => {
    propsValue = value 
})

//子组件需要侦测props
watch(props, (newProps: any, oldProps) => {
    if (newProps.provinceName && newProps.cityName) {
        state.queryAreaInfo[0] = newProps.provinceName
        state.queryAreaInfo[1] = newProps.cityName
    }
}, { immediate: true, deep: true })			//不加immediate会不及时渲染
//需要放在nextTick内的原因是由于Vue的DOm更新不是立马执行的，而是会加入事件循环队列中，propsValue的值再组件加载结束才传过去，如果直接传，也会会导致props的值在组件从事件循环队列开始加载前就已经传的问题。

    # 获取不到Dom上的ref

    原因跟子组件没有渲染props的值一样

    当想清空el-dialog内的form表单，打开后获取ref对象为null，是因为此时获取的ref是在组件的dom加载前便获取（还未绑定Dom），因此打印为空，需要放在nextTick内执行，待下个事件循环再开始。
